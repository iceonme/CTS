# CryptoPulse AI - MVP 功能清单

> 纯功能视角：用户能看到什么、能做什么，不讨论技术实现

---

## MVP 核心定位

一个「加密资产 CFO」助手，帮你：
- **盯盘**：每隔一段时间自动查看你关注的币种
- **提醒**：发现异常或机会时主动告诉你
- **对话**：随时问它市场情况或资产问题

---

## 功能列表（按模块）

### 模块 1：首页 - CFO 对话台

**你能看到的：**
- 一个聊天界面，和 CFO 对话
- CFO 会主动发消息给你（比如"BTC 突破了"、"已完成今日盯盘"）
- 每条 CFO 消息显示：分析结论 + 置信度（0-100%）+ 展开按钮
- 展开后能看到：看多理由 / 看空理由 / 最终判断依据

**你能做的：**
- 发消息问 CFO 问题（"帮我看看 BTC" / "我现在持仓风险大吗"）
- 点击"执行"按钮，确认 CFO 的建议操作（仅在模拟模式下）
- 查看历史对话

**示例对话流：**
```
CFO（自动）: 📊 每日盯盘报告
            BTC: 看涨信号，突破 $67,000
            置信度: 72%
            [查看详情] [好的，知道了]

你: 要不要加仓？

CFO: 建议观望。当前突破伴随放量，但 RSI 接近超买。
     看多理由: 突破阻力位、成交量放大
     看空理由: RSI 72 接近超买、上方有密集成交区
     综合判断: 72% 看涨，建议等待回调至 $65,000 再考虑加仓
     [好的] [设置提醒]
```

---

### 模块 2：市场情报 Feed

**你能看到的：**
- 一个类似 Twitter 的时间线
- 不同颜色的卡片代表不同来源的分析
- 每条情报包含：币种、关键指标、简短解读、时间

**你能做的：**
- 滚动浏览历史情报
- 点击某条情报，查看完整分析
- 按币种筛选（只看 BTC / ETH 相关）

**Feed 内容类型：**
| 类型 | 示例 |
|------|------|
| 技术分析 | BTC 突破 50 日均线，MACD 金叉形成 |
| 异动提醒 | ETH 15 分钟内涨幅 5%，成交量异常放大 3 倍 |
| 定期报告 | 【每日报告】今日市场概览：BTC +3.2%，ETH +1.8% |

---

### 模块 3：WarRoom（决策可视化）

**你能看到的：**
- CFO 的"大脑"：看多模块 vs 看空模块的对战
- 历史正确率统计（CFO 过去 30 天预测对了多少次）
- 最近 10 条决策记录（时间、币种、判断、结果）

**你能做的：**
- 查看某次决策的完整思考过程
- 对比 CFO 预测 vs 实际走势

**可视化概念：**
```
┌─────────────────────────────────────────┐
│  CFO 决策大脑                            │
├─────────────────────────────────────────┤
│  🐂 看多阵营        🐻 看空阵营          │
│  ├─ 突破阻力位      ├─ RSI 超买          │
│  ├─ 成交量放大      └─ 上方密集区        │
│  └─ 情绪偏多                             │
│                                         │
│  综合置信度: [████████░░] 72% 看多      │
│                                         │
│  [查看完整推理过程]                      │
└─────────────────────────────────────────┘
```

---

### 模块 4：资产管理

**你能看到的：**
- 总资产估值（按当前市场价计算）
- 各币种持仓列表（币种、数量、现价、占比）
- 今日盈亏（金额 + 百分比）
- 简单的资产分布饼图

**你能做的：**
- 添加/删除持仓（手动录入你的持仓情况）
- 修改持仓数量
- 查看历史盈亏走势（最近 7 天）

> ⚠️ MVP 阶段：资产数据需要手动录入，不支持连接交易所 API

---

### 模块 5：设置

**你能看到的：**
- CFO 工作设置：盯盘频率（每 X 分钟检查一次）
- 通知设置：是否接收主动消息、异动提醒阈值
- 关注的币种列表

**你能做的：**
- 调整 CFO 检查市场的频率
- 设置价格异动提醒阈值（如：1 小时内涨跌超过 X%）
- 添加/删除关注的币种
- 开启/关闭各类通知

## MVP 不包含的功能（Phase 2）

以下功能先不做，但架构预留扩展空间：

| 不做 | 原因 |
|------|------|
| 真实交易执行 | 需要交易所 API、风控体系，先只做模拟/通知 |
| 连接真实钱包 | 安全门槛高，先手动录入资产 |
| 多智能体独立身份 | 先只做 CFO 一个角色，情报以 CFO 视角呈现 |
| 实时推送 | 先做轮询模式，不增加复杂度 |
| 复杂策略回测 | MVP 先不做历史策略验证 |

## 用户使用流程示例

**Day 1 - 初始化**
1. 打开应用，看到 CFO 欢迎消息
2. 进入设置，添加关注的币种（BTC、ETH）
3. 录入当前持仓（手动输入）
4. 设置盯盘频率（每 30 分钟检查一次）

**Day 2 - 日常运行**
1. CFO 每 30 分钟自动检查市场，无异常时不打扰
2. 15:30，BTC 突然上涨 4%，CFO 发送异动提醒
3. 你打开对话，问 CFO "要追吗？"
4. CFO 给出分析：72% 看多，但建议等回调
5. 你决定观望，点击"设置回调提醒"

**Day 3 - 查看总结**
1. 打开 WarRoom，查看 CFO 最近决策记录
2. 发现 CFO 昨天对 ETH 的看跌判断被验证了
3. 查看 Feed 流，回顾这两天的市场情报
